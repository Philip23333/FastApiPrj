# MyNews 前端项目文档 (Vue 3 + Vite)

## 1. 项目简介
MyNews 前端是一个基于 Vue 3 (Composition API) 搭建的仿今日头条新闻资讯系统。主要包含首页新闻信息流（支持分类切换与分页加载）和新闻详情页（支持浏览量统计与相关新闻推荐）。

## 2. 技术栈
- **框架**: Vue 3.x (使用 `<script setup>`)
- **构建工具**: Vite
- **路由**: Vue Router 4
- **网络请求**: Axios

## 3. 主要页面与组件设计

### 3.1 首页 (Home.vue)
- **核心逻辑**:
  - `fetchCategories()`: 挂载时获取顶部新闻分类标签。
  - `fetchNews(page, append)`: 根据当前选中的分类 (`activeCategoryId`) 获取对应的新闻流。支持"覆盖"与"追加"(用于点击加载更时)。
  - `goToDetail(id)`: 点击新闻卡片时，通过 `router.push('/news/'+id)` 携带对应的新闻ID跳转至详情页。
- **页面布局**:
  - **头部**: 导航栏与搜索框。
  - **左侧边栏**: 独立的分类导航（高亮当前分类）。
  - **中间主区域**: 新闻卡片信息流，展示标题、简介、封面图及作者信息。

### 3.2 新闻详情页 (NewsDetail.vue)
- **核心逻辑**:
  - **路由参数读取**: 通过 `route.params.id` 获取 URL 上的当前文章 ID。利用 `watch` 监听该 ID 的变化，以便在从“相关推荐”跳转到新文章时重新发请求刷新数据及页面。
  - **获取文章详情 (`fetchNewsDetail`)**: 
    - 使用 `axios.get` 访问后台获取文章的详细内容。
    - 服务端触发返回时，后台会自动将该文章的浏览量 (views) +1。
    - 获取成功后，立刻提取该文章的 `category_id` 去请求相关推荐的数据。
  - **相关推荐 (`fetchRelatedNews`)**: 
    - 请求同分类下的多条新闻。
    - 拿到数据后过滤掉当前正在浏览的这篇文章的 ID。
    - 将数组通过 `sort` 打乱并截断（显示四条），提供良好的随机推荐体验。
- **页面布局**:
  - 左侧互动悬浮面板（点赞、收藏等UI）
  - 中间文章主干（标题、作者、发布时间、正文与配图）及底部的分类相关新闻推荐流。

## 4. 重点概念复习
- **响应式数据 (`ref`)**: 如 `const newsItem = ref(null)`，它是响应式的，当被赋予后端接口返回值时，会触发模板（视图）层重新渲染，这就是"数据驱动视图"。
- **箭头函数赋给变量**: 如 `const fetchNewsDetail = async () => {...}`。函数在 JS 中是一等公民，把匿名函数赋值给 `const` 变量可以防止意外覆盖，并处理好作用域内部 `this` 指向的问题。
- **Watch 监听器**: 监听 `route.params.id` 的变动，这在同一个组件内互相跳转（如详情页跳详情页）时非常关键，因为 Vue 为了性能会复用组件，不加监听就不会重新获取数据。
