1. 目标定义（V1）
一键总结当前新闻：在详情页/审核页点击后返回结构化摘要。
发布页智能建议：根据标题、分类、正文草稿给出改写与补全建议。
本地知识库问答：基于库内新闻做检索增强问答（RAG），回答可追溯来源。
2. 总体架构
前端层：新增 AI 助手面板（详情页、发布页、全局问答入口）。
后端层：新增 ai 模块（路由 + 服务 + 提示词 + 模型适配）。
模型层：
生成模型（LLM）：用于总结、建议、问答生成。
向量模型（Embedding）：用于新闻向量化。
知识层：
向量库（建议 Qdrant / pgvector / Milvus 三选一）。
文档切片与召回。
安全层：鉴权、频控、审计日志、输出过滤。
3. 功能拆解
3.1 一键总结当前新闻
输入：news_id 或传入新闻正文。
流程：取新闻内容 -> 组装提示词 -> 调用 LLM -> 返回结构化结果。
输出建议结构：
summary_short：一句话摘要
summary_keypoints：3-5 条要点
risk_tags：敏感/不确定信息标签（可选）
前端展示：卡片式摘要 + 复制按钮 + 重新生成。
3.2 发布页智能建议
输入：标题、分类、正文、描述（可选）。
流程：草稿质量评估 -> 生成建议 -> 返回差异化修改项。
输出建议结构：
title_suggestions
description_suggestion
content_issues（逻辑、重复、长度、口语化）
seo_keywords（可选）
前端交互：右侧 AI 建议抽屉，支持“应用该建议”。
3.3 本地知识库问答（RAG）
数据来源：数据库新闻（优先 approved）。
索引粒度：按新闻正文切片（如 300500 字一块，重叠 5080）。
检索流程：
Query embedding -> TopK 检索 -> 重排（可选）-> 组装上下文 -> 生成答案
输出：
answer
citations（来源新闻标题+id）
confidence（置信度）
前端：聊天面板支持“查看引用新闻”。
4. 数据与索引设计
新增表建议：
ai_chat：会话记录（你库里已有同名表，可扩展）
ai_embedding_job：向量化任务状态
ai_prompt_log：关键请求日志（可选）
向量元数据：
news_id、title、category、publish_time、audit_status
索引更新策略：
新闻新增/修改后异步入队重建向量
审核状态变更时同步索引可见性
删除/下架时从向量库软删除
5. API 大纲（建议）
POST /ai/news/{news_id}/summary
POST /ai/news/draft/suggest
POST /ai/qa
POST /ai/index/rebuild（管理员）
GET /ai/index/status（管理员）
6. 权限与合规
普通用户：可用总结、建议、问答。
reviewer/admin：额外可触发重建索引、查看任务状态。
限流：按用户/分钟请求上限，避免滥用。
脱敏：日志不记录完整正文和密钥。
可追溯：RAG 输出必须附引用来源，降低幻觉风险。
7. 提示词策略（Prompt）
总结：强调“忠于原文、不补事实、输出结构化 JSON”。
建议：强调“建议而非替代，保留原作者表达意图”。
问答：强调“只依据检索片段回答，无法回答时明确说不知道”。
8. 实施里程碑（4 周版）
第 1 周：AI 基础框架 + 总结接口（最小可用）。
第 2 周：发布页建议 + 前端交互打通。
第 3 周：向量化管道 + RAG 问答。
第 4 周：限流、日志、评测、优化与演示稿。
9. 验收标准（面试可展示）
给定任意新闻，2 秒内返回结构化摘要。
发布页可生成至少 3 条有效建议并支持一键应用。
问答结果附可点击来源，答案与来源一致。
下架/拒稿新闻不参与外部问答召回。
系统在并发下不报错，日志可追踪调用链路。
