## 1. 基础信息

- 技术栈：FastAPI
- 应用入口：`App/main.py`
- 路由前缀：
  - 用户模块：`/users`
  - 新闻模块：`/news`
- 跨域：已开启 `allow_origins=["*"]`

## 2. 统一响应结构

所有接口统一返回如下结构（`ApiResponse`）：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

字段说明：

- `code`: 业务状态码（通常与 HTTP 状态码语义一致）
- `message`: 文本消息
- `data`: 业务数据（对象/数组/null）

### 2.1 分页响应 data 结构

列表分页接口的 `data` 结构：

```json
{
  "total": 100,
  "page": 1,
  "size": 10,
  "totalPages": 10,
  "items": []
}
```

### 2.2 全局异常响应

- `HTTPException`：返回 `code/message/data`
- `RequestValidationError`（422）：

```json
{
  "code": 422,
  "message": "Validation Error",
  "data": [
    {
      "loc": ["body", "password"],
      "msg": "Field required",
      "type": "missing"
    }
  ]
}
```

## 4. 用户接口（/users）

## 4.1 用户注册

- 方法：`POST`
- 路径：`/users/register`
- HTTP 状态：`201`
- 请求体：

```json
{
  "username": "string(必填, <=50)",
  "password": "string(必填, <=255)",
  "nickname": "string(可选, <=50)",
  "avatar": "string(可选, <=255)",
  "gender": "male|female|unknown(可选)",
  "bio": "string(可选, <=500)",
  "phone": "string(可选, <=20)"
}
```

- 成功响应：

```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "alice",
    "nickname": "Alice",
    "avatar": null,
    "gender": "unknown",
    "bio": null,
    "phone": null,
    "created_at": "2026-03-25T10:00:00",
    "updated_at": "2026-03-25T10:00:00"
  }
}
```

- 失败场景：
  - 用户名已存在：`400`，`message=用户名已被注册`

## 4.2 用户登录

- 方法：`POST`
- 路径：`/users/login`
- 请求体：

```json
{
  "username": "string(必填)",
  "password": "string(必填)"
}
```

- 成功响应：`200`，返回用户信息
- 失败场景：
  - 账号或密码错误：`400`，`message=用户名或密码错误`
  - 缺字段：`422`（Validation Error）

## 4.3 用户列表

- 方法：`GET`
- 路径：`/users/`
- Query 参数：
  - `skip`：默认 `0`
  - `limit`：默认 `100`
- 成功响应：`200`，`data` 为用户数组

## 4.4 用户详情

- 方法：`GET`
- 路径：`/users/{user_id}`
- Path 参数：
  - `user_id`：用户 ID（int）
- 成功响应：`200`
- 失败场景：
  - 用户不存在：`404`，`message=User not found`

## 4.5 更新用户

- 方法：`PUT`
- 路径：`/users/{user_id}`
- Path 参数：
  - `user_id`：用户 ID（int）
- 请求体（全部可选）：

```json
{
  "nickname": "string",
  "avatar": "string",
  "gender": "string",
  "bio": "string",
  "phone": "string",
  "password": "string"
}
```

- 成功响应：`200`，返回更新后的用户信息
- 失败场景：
  - 用户不存在：`404`，`message=User not found`

## 4.6 删除用户

- 方法：`DELETE`
- 路径：`/users/{user_id}`
- Path 参数：
  - `user_id`：用户 ID（int）
- 成功响应：`200`，`data=null`
- 失败场景：
  - 用户不存在：`404`，`message=User not found`

## 5. 新闻接口（/news）

## 5.1 新闻分页列表

- 方法：`GET`
- 路径：`/news/`
- Query 参数：
  - `page`：默认 `1`
  - `size`：默认 `10`
- 成功响应：`200`，分页结构（`data.total/page/size/totalPages/items`）
- `items[]` 字段：
  - `id`
  - `title`
  - `description`
  - `category_name`
  - `author`
  - `views`
  - `publish_time`
  - `image`

## 5.2 热榜新闻（支持换一换）

- 方法：`GET`
- 路径：`/news/hot`
- Query 参数：
  - `min_views`：浏览量阈值，默认 `5000`
  - `page`：批次页码，默认 `1`
  - `size`：每批条数，默认 `8`
- 成功响应：`200`
- `data`：数组，元素字段为 `id/title/views`
- 特殊语义：
  - 当 `page>1` 且无更多数据时：返回 `code=200`，`message="reached_end"`，`data=[]`

## 5.3 新闻详情

- 方法：`GET`
- 路径：`/news/detail/{news_id}`
- Path 参数：
  - `news_id`：新闻 ID（int）
- 成功响应：`200`，返回完整新闻对象
- 详情字段：
  - `id`
  - `title`
  - `content`
  - `category_id`
  - `category_name`
  - `author`
  - `views`
  - `publish_time`
  - `image`
- 失败场景：
  - 新闻不存在：`404`，`message=News not found`
- 注意：该接口包含副作用，会将新闻浏览量 `views + 1`

## 5.4 新闻分类列表

- 方法：`GET`
- 路径：`/news/categories`
- Query 参数：
  - `skip`：默认 `0`
  - `limit`：默认 `100`
- 成功响应：`200`，`data` 为分类数组
- 分类元素常见字段：`id`, `name`, `sort_order`, `created_at`, `updated_at`

## 5.5 分类下新闻列表

- 方法：`GET`
- 路径：`/news/categories/{category_id}/news`
- Path 参数：
  - `category_id`：分类 ID（int）
- Query 参数：
  - `page`：默认 `1`
  - `size`：默认 `10`
- 成功响应：`200`，分页结构同新闻列表
- `items[]` 字段同 `/news/`

## 6. 状态码总览

- `200`：成功
- `201`：创建成功（注册）
- `400`：业务错误（例如用户名重复、登录失败）
- `404`：资源不存在（用户/新闻）
- `422`：请求参数校验失败
