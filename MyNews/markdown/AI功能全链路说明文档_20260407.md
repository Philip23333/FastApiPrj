# MyNews AI 功能全链路说明文档

## 1. 文档目标

本文面向新同学，完整说明当前项目 AI 模块的实现方式与代码入口，覆盖以下内容：

- 后端 AI 接口设计与参数
- 模型调用链路（LangChain + 兼容 OpenAI 协议）
- Schema 定义与返回结构
- Prompt 设计
- RAG 向量库构建与增量同步
- 前端三大 AI 功能接入方式
- 常见问题与排查手册

## 2. 架构总览

AI 模块由 4 层组成：

1. 路由层：统一暴露 AI API
2. 业务层：摘要、写作建议、RAG 问答编排
3. 模型层：统一调用 LLM / Embedding
4. 存储层：Qdrant 向量索引（全量重建 + 增量同步）

核心入口文件：

- [App/main.py](../App/main.py#L90) 挂载 AI 路由
- [App/routers/ai.py](../App/routers/ai.py) AI 接口
- [App/services/ai/service.py](../App/services/ai/service.py) 摘要/写作建议服务
- [App/services/ai/client.py](../App/services/ai/client.py) 模型统一客户端
- [App/services/ai/prompts.py](../App/services/ai/prompts.py) Prompt 模板
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py) RAG 核心
- [App/schemas/ai.py](../App/schemas/ai.py) AI 请求/响应模型

统一响应格式：`ApiResponse[T]`，通过 `success_response` 返回。

### 3.1 通用对话接口

接口：POST `/ai/chat`

请求参数（对应 [App/schemas/ai.py](../App/schemas/ai.py#L6) `AIChatIn`）：

- `message: str` 必填，1~4000
- `system_prompt: str | null` 可选，最大 2000
- `temperature: float` 可选，默认 0.3，范围 [0, 1]

响应参数（对应 [App/schemas/ai.py](../App/schemas/ai.py#L31) `AIChatOut`）：

- `answer: str`
- `model: str`

### 3.2 新闻 AI 总结

接口：POST `/ai/news/{news_id}/summary`

逻辑入口：

- [App/routers/ai.py](../App/routers/ai.py#L46) `summarize_news`
- [App/services/ai/service.py](../App/services/ai/service.py#L19) `AIService.summarize_news`

请求参数：

- 路径参数 `news_id: int`

响应参数（对应 [App/schemas/ai.py](../App/schemas/ai.py#L19) `AINewsSummaryOut`）：

- `news_id: int`
- `summary: str`
- `key_points: list[str]`

实现说明：

- 先查新闻正文，若不存在或删除则 404
- 调用 LLM 总结
- 对模型返回做容错拆分：第 1 行作为 `summary`，后续最多 3 行作为 `key_points`

### 3.3 写作助手建议

接口：POST `/ai/news/draft/suggest`

逻辑入口：

- [App/routers/ai.py](../App/routers/ai.py#L72) `suggest_news_draft`
- [App/services/ai/service.py](../App/services/ai/service.py#L33) `AIService.suggest_news_draft`

请求参数（对应 [App/schemas/ai.py](../App/schemas/ai.py#L12) `AINewsDraftSuggestIn`）：

- `title: str` 必填，1~120
- `category_name: str` 必填，1~50
- `content: str` 必填
- `description: str | null` 可选，最大 300

响应参数（对应 [App/schemas/ai.py](../App/schemas/ai.py#L25) `AINewsDraftSuggestOut`）：

- `title_suggestions: list[str]`
- `description_suggestion: str | null`
- `content_suggestions: list[str]`

解析策略（强容错）：

1. 优先按 PydanticOutputParser 解析结构化 JSON
2. 失败后尝试 `json.loads`
3. 仍失败则按纯文本分行回退

### 3.4 RAG 问答

接口：POST `/ai/qa`

逻辑入口：

- [App/routers/ai.py](../App/routers/ai.py#L88) `qa_with_rag`
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L435) `answer_question`

请求参数（对应 [App/schemas/ai.py](../App/schemas/ai.py#L36) `AIQAIn`）：

- `question: str` 必填，1~2000
- `top_k: int` 可选，默认 4，范围 [1, 10]

响应参数（对应 [App/schemas/ai.py](../App/schemas/ai.py#L48) `AIQAOut`）：

- `answer: str`
- `model: str`
- `citations: list[AIQACitationOut]`

引用字段（[App/schemas/ai.py](../App/schemas/ai.py#L41)）：

- `news_id: int`
- `title: str`
- `snippet: str`
- `score: float`

特殊逻辑：

- 首次调用若索引为空，会自动触发一次全量构建

### 3.5 索引管理接口

- POST `/ai/rag/index/rebuild`：手动全量重建（管理员/审核员）
- GET `/ai/rag/index/status`：查看索引状态

状态模型（[App/schemas/ai.py](../App/schemas/ai.py#L54)）：

- `indexed_news_count`
- `indexed_chunk_count`
- `embedding_model`
- `chunk_size`
- `chunk_overlap`
- `last_rebuild_at`

## 6. RAG 向量库构建与检索

核心文件：[App/services/ai/rag_service.py](../App/services/ai/rag_service.py)

### 6.1 关键方法

- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L50) provider 批量上限
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L56) 文档 embedding（自动降批重试）
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L88) query embedding
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L174) HTML/文本清洗
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L216) 全量重建
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L327) 增量同步
- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L435) 检索问答

### 6.2 全量构建流程

1. 查询可索引新闻（已审核通过 + 未删除）
2. 清洗文本（去 script/style/tag/实体，压缩空白）
3. 按 chunk_size/chunk_overlap 切片
4. 调用 embedding 接口，自动按 provider 上限分批
5. 重建 collection 并写入点位
6. 更新状态信息

### 6.3 增量同步流程（非常关键）

`sync_news_index(db, news_id)` 策略：

- 新闻不存在/已删除/非 approved：从向量库删除该 news_id 全部点位
- 新闻 approved：重新切片 + embedding + upsert

点位 ID 策略：

- 增量点位 ID 使用稳定 UUID（uuid5）
- 这是为了解决 Qdrant 只接受无符号整数或 UUID 的约束

### 6.4 索引自动维护触发点

新闻主流程中的触发位置：

- [App/routers/news.py](../App/routers/news.py#L243) 作者更新后（通常转 pending，移除旧索引）
- [App/routers/news.py](../App/routers/news.py#L352) 删除新闻后（删除索引）
- [App/routers/news.py](../App/routers/news.py#L500) 审核动作后（approved upsert，其他状态删除）

## 9. 排查手册（新人常见）

### 9.1 AI 接口报 AI 功能未启用

现象：返回“AI 功能未启用，请设置 AI_ENABLED=true”

排查：

1. 检查 `AI_ENABLED`
2. 检查 `AI_API_KEY`
3. 重启服务确认环境变量已生效

### 9.2 RAG 问答报索引为空

现象：`RAG 索引为空，请先执行索引构建`

排查：

1. 调 `/ai/rag/index/status`
2. 手动调用 `/ai/rag/index/rebuild`

### 9.3 新闻审核通过后未入库

排查：

1. 看 [App/routers/news.py](../App/routers/news.py#L500) 是否执行到增量同步
2. 看 RAG 日志是否出现 embedding 失败
3. 看 Qdrant 配置和 collection 维度是否一致

### 9.4 来源文本出现异常断词

根因：文本清洗正则错误会误删字母。

当前修复点：

- [App/services/ai/rag_service.py](../App/services/ai/rag_service.py#L174)

若历史索引已受污染：

- 需要重新执行全量重建，旧脏数据不会自动自愈
