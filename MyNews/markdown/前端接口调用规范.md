# 前端接口调用规范（MyNews）

更新时间：2026-03-26

## 1. 目标

统一前端对后端 API 的调用方式，降低联调成本，避免鉴权、分页、错误处理不一致带来的问题。

## 2. 基础约定

- 前端默认 API 基地址：`http://127.0.0.1:8080`
- 建议通过环境变量配置：`VITE_API_BASE_URL`
- 当前项目统一拼接函数：`withApiBase(path)`

建议：

- 开发环境：`.env.development` 设置 `VITE_API_BASE_URL`
- 生产环境：`.env.production` 设置真实网关地址

## 3. 统一响应判定

后端统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

前端处理规则：

1. HTTP 状态码非 2xx：走 `catch`。
2. HTTP 2xx 但 `code` 非 200/201：按业务失败处理。
3. 成功数据一律从 `res.data.data` 读取。

## 4. 鉴权规范

- 登录成功后将 `access_token` 存入本地（如 `localStorage.accessToken`）。
- 受保护接口统一自动注入：

```http
Authorization: Bearer <token>
```

- 401 统一处理：清除登录态并跳转登录/首页。
- 403 统一处理：提示“无权限或账号已禁用”。

当前项目已在全局 axios 拦截器实现自动带 token 与 401 处理，页面层不应重复拼接 Authorization。

## 5. 分页规范

### 5.1 后端分页参数

- 通用入参：`page`、`size`
- 通用返回：`data.total`、`data.page`、`data.size`、`data.totalPages`、`data.items`

### 5.2 前端调用要求

- `page` 从 1 开始。
- `size` 建议遵守后端上限（常见上限 50 或 100）。
- 不要假设 `items.length < size` 就一定到末页，应结合 `total` 与当前偏移判断。

## 6. 新闻审核流相关约定

1. 公共列表、分类、搜索只展示 `approved` 新闻。
2. `rejected` 或删除新闻详情不可访问（404）。
3. 作者编辑并提交后，新闻会重置为 `pending`，需再次审核通过才会出现在首页。
4. 个人中心作品列表优先使用 `/news/mine`，不要用公共列表替代。
5. 编辑页回填优先使用 `/news/editable/{news_id}`，保证拒稿内容可继续编辑。

## 7. 下架态展示规范（历史记录）

`/history` 返回中如果 `is_removed = true`：

- 标题显示“新闻已下架”。
- 描述显示“该新闻已下架，暂不可查看详情。”。
- 前端禁用跳转到详情页。

## 8. 上传接口规范

- 上传地址：`POST /files/upload`
- 请求体：`multipart/form-data`
- 字段名：`file`
- 文件类型：`image/*`
- 成功读取：`res.data.data.url`

示例：

```js
const fd = new FormData()
fd.append('file', file)
const res = await axios.post(withApiBase('/files/upload'), fd)
const imageUrl = res?.data?.data?.url || ''
```

## 9. 错误处理规范

建议封装统一错误映射：

- 400：参数或业务校验失败，展示后端 message。
- 401：登录失效，跳转登录。
- 403：无权限或账号禁用。
- 404：资源不存在或已下架。
- 422：表单字段校验失败，提示“请检查输入项”。
- 500：服务异常，提示“服务开小差，请稍后再试”。

页面层建议：

- 同一操作避免重复 toast。
- 管理页面提交按钮加 loading，避免重复提交。

## 10. 接口组织建议

建议按业务拆分调用层（可逐步改造）：

- `src/api/user.js`
- `src/api/news.js`
- `src/api/favorite.js`
- `src/api/history.js`
- `src/api/upload.js`

每个 API 函数只做两件事：

1. 拼接 URL 与参数。
2. 返回后端原始响应或已解包的 `data`。

页面层负责展示逻辑，不直接拼接复杂 URL 字符串。

## 11. 调用示例（推荐写法）

```js
import axios from 'axios'
import { withApiBase } from '@/config/api'

export async function fetchMyNews(page = 1, size = 20) {
  const res = await axios.get(withApiBase('/news/mine'), { params: { page, size } })
  return res?.data?.data
}

export async function reviewNews(newsId, payload) {
  const res = await axios.patch(withApiBase(`/news/admin/${newsId}/moderation`), payload)
  return res?.data?.data
}
```

## 12. 联调检查清单

1. 是否使用 `withApiBase` 统一拼接基础地址。
2. 是否依赖全局拦截器自动注入 Bearer token。
3. 是否按 `res.data.data` 读取业务数据。
4. 是否正确处理 401/403/404/422。
5. 分页是否基于 `total` 和 `totalPages` 计算翻页状态。
6. 是否实现下架态（`is_removed`）禁点与文案展示。
